# Plan: Revise Phase 3 Architecture — React Static Frontend via CDN

## Context

The current architecture uses FastAPI + Jinja2 + HTMX for server-rendered HTML. The revision changes the web layer to:

- **Static HTML page** with React loaded via CDN (no build step, no Node.js tooling)
- **FastAPI serves everything** — static files + JSON API endpoints (single process)
- **No Jinja2, no HTMX** — React handles all rendering and interactivity client-side
- **JSON API** — FastAPI returns data as JSON, React fetches and renders

This contradicts ADR-001 which rejected \"React SPA + API backend\" — so we'll need to supersede that ADR.

## Implementation Steps

### 1. Update `docs/03-architecture-data/Architecture-Overview.md`

Changes:
- **Boundaries & Ownership table**: Update Web App description from \"Filtered view rendering\" to \"JSON API + static file serving\"
- **Project Structure**: Replace `templates/` directory with `static/` directory containing `index.html`, `app.js` (React components), and `styles.css`
- **Web App Routes**: Change from HTML-returning routes to JSON API:
  - `GET /` → serves `static/index.html`
  - `GET /api/movies` → JSON list (filtered, grouped by year)
  - `POST /api/movies/{id}/read` → JSON response
  - `POST /api/movies/{id}/unread` → JSON response
  - `POST /api/movies/{id}/enrich` → JSON response
  - `GET /api/health` → JSON feed health status
- **Technology Stack**: Remove Jinja2 and HTMX, add React (CDN), keep FastAPI
- **Section 2 Interfaces**: Update response types from \"HTML\" / \"HTMX partial\" to \"JSON\"

### 2. Update `docs/03-architecture-data/Data-Contracts.md`

Changes:
- **No schema changes** to database models (Movie, FeedHealth remain the same)
- **Add Section 7: API Response Contracts** — define JSON response shapes for each endpoint:
  - `GET /api/movies` response shape (array of movie objects, grouped by year)
  - `POST /api/movies/{id}/read` response shape
  - `POST /api/movies/{id}/enrich` response shape
  - `GET /api/health` response shape

### 3. Create/Update ADR: `docs/adr/ADR-004-react-cdn-frontend.md`

- **Status:** Accepted
- **Supersedes:** ADR-001 (partially — keeps FastAPI decision, reverses Jinja2/HTMX choice)
- **Decision:** Use React via CDN for client-side rendering, removing Jinja2 server-rendered templates and HTMX
- **Rationale:** User preference for React-based interactivity while keeping the stack simple (no build step, CDN-loaded)
