### Implementation Steps

#### Step 1: Project scaffolding
- Create `requirements.txt` with: fastapi, uvicorn, sqlalchemy, pymysql, feedparser, httpx, pyyaml, python-dotenv
- Create `config.yaml` with full config schema from Data Contracts (DB URL, feed URL, alerting, enrichment, filtering, genre_priority, webapp)
- Create `src/common/__init__.py`, `src/cli/__init__.py`, `src/webui/__init__.py`

#### Step 2: Shared layer (`src/common/`)
- **`src/common/config.py`** — Load + validate `config.yaml` (PyYAML), expose as a dataclass/dict. Handle missing file gracefully.
- **`src/common/models.py`** — SQLAlchemy 2.0 declarative models: `Movie`, `FeedHealth` (exact schema from Data Contracts)
- **`src/common/db.py`** — Engine factory from config URL, session maker, `init_db()` that calls `create_all()`

#### Step 3: CLI Ingester (`src/cli/`)
- **`src/cli/fetcher.py`** — Fetch RSS via feedparser, parse `<title>` (regex for title/year/quality), parse `<description>` HTML (poster, genre, IMDb from feed), return list of parsed movie dicts
- **`src/cli/dedup.py`** — Given a parsed movie dict + DB session: check `torrent_url` match → merge qualities; else check `title+year` → merge; else insert new
- **`src/cli/alerter.py`** — Check `FeedHealth.last_success_at` vs threshold (24h). If exceeded and no recent alert, send email via `smtplib`. Update `alert_sent_at`.
- **`src/cli/main.py`** — Entry point: load config, init DB, fetch RSS, deduplicate/store each entry, update feed_health (success/failure), run alert check, log summary, exit

#### Step 4: Web UI Backend (`src/webui/`)
- **`src/webui/app.py`** — FastAPI app factory: mount `/static` directory, include router from `routes.py`
- **`src/webui/routes.py`** — Endpoints:
  - `GET /` → serve `index.html`
  - `GET /api/movies` → filtered, grouped by year, ordered by genre priority
  - `POST /api/movies/{id}/read` → toggle is_read=True
  - `POST /api/movies/{id}/unread` → toggle is_read=False
  - `POST /api/movies/{id}/enrich` → trigger enrichment
  - `GET /api/health` → feed health status
- **`src/webui/filters.py`** — Apply config-driven filtering: default thresholds, genre-specific overrides, older_movies stricter rules. Group by year sections (2026→2021 + "Older"). Sort by genre priority within sections.
- **`src/webui/enrichment.py`** — Call free rating API (OMDb free tier or imdbapi.dev). Validate returned ratings (V-006..V-008). Update movie record. Handle timeout/errors gracefully.
- **`src/webui/main.py`** — Entry point: load config, init DB, create app, run Uvicorn

#### Step 5: React Frontend (`src/webui/static/`)
- **`index.html`** — HTML shell with CDN script tags for React 18 + Babel Standalone + `app.js`
- **`app.js`** — React components (JSX):
  - `App` — fetches `/api/movies`, renders year sections
  - `MovieCard` — displays movie info (title, year, genres, qualities, ratings, poster), "mark as read" button, "refresh ratings" button
  - `HealthBanner` — fetches `/api/health`, shows status
- **`styles.css`** — Clean layout: card grid, year section headers, responsive

#### Step 6: Logging
- Use Python `logging` with structured format (timestamp, component, level, message)
- CLI logs to stdout (cron captures it)
- WebUI logs to stdout (Uvicorn default)

#### Step 7: `.gitignore`
- Ignore `config.yaml` (contains secrets), `*.db`, `__pycache__`, `.env`
